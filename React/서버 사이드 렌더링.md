# 서버 사이드 렌더링<br>  
## 서버 사이드 렌더링이란?<br>  
  
### 싱글 페이지 애플리케이션**이란?**<br>  
싱글 페이지 애플리케이션(Single Page Application: SPA)이란 렌더링과 라우팅에 필요한 대부분의 기능을 서버가 아닌 브라우저의 자바스크립트에 의존하는 방식을 의미한다.  
  
* 최초의 첫 페이지에서 데이터를 모두 불러온다.  
* 페이지 전환을 위한 모든 작업이 자바스크립트와 브라우저 `history.pushState`와 `history.replaceState`로 이루어진다.  
* 불러온 이후에는 서버에서 HTMl을 내려받지 않고 하나의 페이지에서 모든 작업을 처리한다.  
  
### **장점**<br>  
* **빠른 화면 전환**: 페이지 전체를 새로고침하지 않고 필요한 데이터만 로딩하므로 사용자 경험이 부드럽고 빠름  
* **클라이언트 중심 라우팅**: 사용자 행동에 따라 유연하게 UI 변경 가능  
* **프론트엔드 개발 효율성**: 컴포넌트 단위 개발과 상태 관리가 용이함  
### **단점**<br>  
* **초기 로딩 속도**: 모든 리소스를 한 번에 불러오므로 초기 진입이 느릴 수 있음  
* **SEO 어려움**: 클라이언트 렌더링 특성상 검색 엔진 최적화가 까다로움  
* **복잡한 상태 관리**: 앱 규모가 커질수록 클라이언트 상태 관리가 어려워질 수 있음  
  
### 서버 사이드 렌더링이란?<br>  
서버 사이드 렌더링(Server-Side Rendering: SSR)이란 최초의 사용자에게 보여줄 페이지를 서버에서 렌더링해 빠르게 사용자에게 화명을 제공하는 방식이다.  
  
* 렌더링에 필요한 작업을 모두 서버에서 수행한다.  
* 클라이언트의 렌더링은 사용자 기기의 성능에 영향을 받지만 서버 사이드 렌더링은 서버에서 제공하기 때문에 비교적 안정적인 렌더링이 가능하다.  
  
### 장점<br>  
* **SEO에 유리함**: 초기 HTML이 서버에서 완성되어 전달되므로 검색 엔진이 쉽게 크롤링 가능  
* **초기 렌더링 속도 개선**: 클라이언트보다 서버가 더 빠르게 초기 화면을 렌더링할 수 있음  
* **소셜 미디어 미리보기 지원**: 메타 태그가 HTML에 포함되어 공유 시 미리보기가 정확히 출력됨  
### 단점<br>  
* **서버 부하 증가**: 렌더링을 서버에서 처리하므로 트래픽이 많을수록 서버에 부담  
* **개발 복잡도 증가**: 클라이언트/서버에서 모두 동작하는 코드를 신경 써야 함  
* **응답 지연 가능성**: 서버 렌더링 시간이 길어지면 TTFB(Time to First Byte)가 느려질 수 있음  
  
### SPA와 SSR을 모두 알아야 하는 이유<br>  
1. 다수의 사용자에게 좋은 사용자 경험을 제공할 수 있는 웹 애플리케이션을 만들고 싶다면 두 가지 방법을 모두 숙지할 필요가 있다.  
1. 필요에 따라 맞는 것을 선택해야한다.
